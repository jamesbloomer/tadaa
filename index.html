Tadaa!

Monitor stuff via the medium of sound

Why use Information Radiators?

Communication. Information Radiators highlight key events, show important data and let you summarise the state of a team or project concisely and clearly.

Why use Tadaa?

Tadaa is an audio information radiators. Use it to play a sounds when your build fails, when you push your app live, when your fix a bug or any other events you feel like. Set Tadaa running and sounds will alert you to important events without the need to glance at another screen or gaze across at a wall of post-it notes.

Quickstart

Install

Install the runner:

npm install tadaa-runner

Now install any required plugins:

npm install tadaa-zendesk --save

Copy any sound files required to the root directory of tadaa-runner (issue [#1](https://github.com/jamesbloomer/tadaa-runner/issues/1)).

Configure

Copy the example-config.js installed with tadaa-runner to config.js. Edit config.js. See the plugins documentation for values to configure.

Run

./node_modules/tadaa-runner/bin/tadaa

Plugins

Plugins for Tadaa include:

  • Pivotal Tracker
  • Zendesk
  • Thoughtworks Go
  • AWS Elastic Load Balancer

Further Documentation

Tadaa consists of the core node module, a command line runner and a set of plugins.
